# Quarkus Patterns (Cloud-Native Java)

## Overview
Quarkus is a modern, Kubernetes-native Java framework tailored for GraalVM and HotSpot. If Spring Boot is the massive enterprise default, Quarkus is the high-performance, low-memory alternative designed to rival Node.js and Go in startup speed and resource usage.

**For a Node.js developer who values fast startup, low RAM footprint, and instant hot-reloading, Quarkus is highly recommended.**

## Why Quarkus?
1. **Live Coding**: Instant hot-reload. Just save the file, and the changes are instantly reflected without restarting the server manually (similar to `nodemon` or `ts-node-dev`).
2. **Native Compilation**: Using GraalVM, Quarkus compiles Java ahead-of-time (AOT) into a native executable binary.
   - *Result*: ~15ms startup time and ~30MB RAM usage (compared to Spring's 3 seconds and 200MB+ RAM).
3. **Standards-Based**: It uses standard Java APIs (Jakarta EE, MicroProfile, Hibernate) rather than framework-specific magic.

## Bootstrapping
Similar to Spring Initializr, Quarkus has [code.quarkus.io](https://code.quarkus.io/).
Or via the Quarkus CLI:
```bash
quarkus create app com.mycompany:my-app
cd my-app
quarkus dev # Starts Live Coding mode!
```

## Recommended Project Structure
```text
src/main/java/com/mycompany/
│
├── user/                     # Feature-based grouping (Domain Driven Design)
│   ├── UserResource.java     # Endpoints (JAX-RS)
│   ├── UserService.java      # Business logic
│   └── UserEntity.java       # DB Entity (Panache)
└── config/                   # Global configs
```

## Routing (JAX-RS)
Quarkus uses standard JAX-RS annotations for routing instead of Spring annotations.

```java
import jakarta.ws.rs.*;
import jakarta.ws.rs.core.MediaType;

@Path("/api/v1/users")
@Produces(MediaType.APPLICATION_JSON)
@Consumes(MediaType.APPLICATION_JSON)
public class UserResource {

    @Inject
    UserService userService;

    @GET
    @Path("/{id}")
    public User getUser(@PathParam("id") Long id) {
        return userService.findById(id);
    }

    @POST
    public Response createUser(UserCreateDTO dto) {
        User user = userService.create(dto);
        return Response.status(Response.Status.CREATED).entity(user).build();
    }
}
```

## Hibernate with Panache (Active Record Pattern)
Quarkus simplifies database access immensely with "Panache", which brings the **Active Record** pattern (similar to Laravel Eloquent or TypeORM) to Java. This greatly reduces repository boilerplate.

```java
import io.quarkus.hibernate.orm.panache.PanacheEntity;
import jakarta.persistence.Entity;

@Entity
public class User extends PanacheEntity {
    // The "id" field is automatically generated by PanacheEntity!
    public String name;
    public String email;
    
    // Custom database queries can be placed directly in the entity
    public static User findByEmail(String email){
        return find("email", email).firstResult();
    }
}
```

Usage in a Service layer:
```java
@ApplicationScoped
public class UserService {
    
    @Transactional
    public void createUser() {
        User user = new User();
        user.name = "Alice";
        user.email = "alice@example.com";
        user.persist(); // Saves directly to DB!
    }
}
```

## Native Compilation
To compile to a tiny, blazing-fast native binary (requires GraalVM locally or using Docker):
```bash
./mvnw package -Dnative
```
This produces a standalone executable that **does not require a JVM** to run!
